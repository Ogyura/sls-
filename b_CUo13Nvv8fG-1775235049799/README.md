# Regional Analytics Platform

Система мониторинга региональных медиа с автоматическим сбором данных из социальных сетей и новостных источников.

## Описание проекта

Платформа для анализа региональных медиа-пространства. Система автоматически собирает публикации из Telegram, VK, веб-сайтов и RSS-лент, анализирует тональность сообщений, выявляет тренды и формирует алерты по критическим событиям.

**Основные функции:**
- Автоматический сбор данных из социальных сетей
- AI-анализ тональности сообщений (positive/negative/neutral)
- Обнаружение трендов и вирусного контента
- Система алертов по критическим событиям
- Гео-аналитика по районам и городам
- Панель администратора
- Импорт/экспорт данных в JSON

## Архитектура базы данных

### Основные таблицы

**data_sources** — Источники данных (Telegram-каналы, VK-группы, веб-сайты)
- Поля: название, тип источника, URL, регион, статус парсинга
- Настройки: интервал обновления, приоритет, последнее обновление

**social_posts** — Собранные публикации
- Контент: текст, заголовок, краткое описание
- Метаданные: автор, дата публикации, URL
- Аналитика: тональность (sentiment), оценка (-1.0 до 1.0), категория
- Engagement: просмотры, лайки, комментарии, репосты
- Гео-данные: регион, координаты (lat/lng)
- Вирусность: флаг тренда, вирусный score

**post_comments** — Комментарии к публикациям
- Связь с родительским постом
- Текст, автор, статистика лайков
- Анализ тональности комментария

**alerts** — Система уведомлений
- Типы: критичные, высокие, средние, низкие
- Статусы: новый, в работе, решён, проигнорирован
- Связь с триггерным постом
- Правило, которое вызвало алерт

**parsing_logs** — Логи производительности
- Время начала и окончания парсинга
- Количество обработанных элементов
- Длительность выполнения
- Ошибки и детали

**parsing_rules** — Правила автоматической обработки
- Условия: ключевые слова, тональность, категория
- Действия: создание алерта, уведомление
- Приоритет и статус активности

## Установка и настройка

### Предварительные требования

Необходимо установить:
- Node.js версии 18 или выше
- Python версии 3.11 или выше
- PostgreSQL версии 14 или выше

### Настройка базы данных

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE regional_analytics;
```

Или через командную строку:
```bash
createdb regional_analytics
```

### Установка Backend

Перейдите в директорию backend:
```bash
cd backend
```

Создайте виртуальное окружение Python:
```bash
python -m venv venv
```

Активируйте окружение:
- Windows: `venv\Scripts\activate`
- Linux/Mac: `source venv/bin/activate`

Установите зависимости:
```bash
pip install -r requirements.txt
```

Основные зависимости backend:
- fastapi — веб-фреймворк
- uvicorn — ASGI-сервер
- sqlalchemy — ORM
- asyncpg — драйвер PostgreSQL
- aiohttp — HTTP-клиент
- beautifulsoup4 — HTML-парсер
- feedparser — RSS-парсер
- pydantic — валидация данных
- python-dotenv — работа с .env файлами

Создайте файл конфигурации `.env`:
```
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/regional_analytics
DEBUG=true
LOG_LEVEL=INFO
```

### Установка Frontend

Вернитесь в корневую директорию:
```bash
cd ..
```

Установите npm-зависимости:
```bash
npm install
```

Основные зависимости frontend:
- next — фреймворк
- react, react-dom — библиотека UI
- typescript — типизация
- tailwindcss — стилизация
- @radix-ui/* — примитивы UI (дропдауны, чекбоксы, скролл)
- lucide-react — иконки
- clsx, tailwind-merge — утилиты для классов

## Запуск проекта

### Запуск Backend

Активируйте виртуальное окружение (если не активно):
```bash
cd backend
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac
```

Запустите сервер:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Сервер будет доступен:
- API: http://localhost:8000
- Документация Swagger: http://localhost:8000/docs
- Health check: http://localhost:8000/health

### Запуск Frontend

В новом терминале, из корневой директории:
```bash
npm run dev
```

Или для production-сборки:
```bash
npm run build
npm start
```

Приложение будет доступно: http://localhost:3000

## Система парсинга

### Автоматический сбор данных

**Расписание парсинга:** 1 раз в день, ночью (1440 минут = 24 часа)

Система работает в фоновом режиме и автоматически:
1. Проверяет все активные источники по расписанию (раз в сутки ночью)
2. Парсит новые публикации из Telegram, VK, веб-сайтов
3. Анализирует тональность каждого сообщения
4. Определяет категорию (транспорт, ЖКХ, здравоохранение и др.)
5. Создаёт алерты при совпадении правил
6. Сохраняет всё в базу данных PostgreSQL

### Источники данных (предустановленные)

При первом запуске автоматически создаются:
- 4 Telegram-канала (РостовГазета, 161.ru, Типичный Ростов, Ростов Сейчас)
- 2 VK-группы (ВК Ростов Главный, Подслушано Ростов)
- 2 Веб-сайта (Дон-ТР, Ростовские Ведомости)
- 3 Региональных источника (Таганрог, Шахты, Новочеркасск)

### Анализ тональности

Встроенный анализатор на основе ключевых слов:
- **Позитивные**: хороший, отличный, спасибо, успех, поддержка
- **Негативные**: плохой, авария, проблема, жалоба, коррупция
- Расчёт score от -1.0 (негатив) до +1.0 (позитив)

### Категоризация

Автоматическое определение темы:
- Транспорт (дороги, автобусы, пробки)
- ЖКХ (вода, свет, отопление)
- Здравоохранение (больницы, врачи)
- Образование (школы, вузы)
- Безопасность (аварии, ДТП, преступления)
- Экономика (работа, цены, бизнес)
- Культура (концерты, театры, выставки)
- Экология (загрязнение, мусор, заводы)

## API Endpoints

### Публикации
- `GET /api/v1/posts` — список с фильтрацией (категория, регион, сентимент)
- `GET /api/v1/posts/trending` — трендовые публикации
- `GET /api/v1/categories` — список категорий
- `GET /api/v1/regions` — список регионов

### Дашборд
- `GET /api/v1/dashboard/stats` — общая статистика
- `GET /api/v1/dashboard/timeline` — таймлайн активности

### Алерты
- `GET /api/v1/alerts` — список алертов
- `PUT /api/v1/alerts/{id}/resolve` — отметить как решённый

### Источники
- `GET /api/v1/sources` — список источников
- `POST /api/v1/sources/{id}/parse` — ручной запуск парсинга

## Разработка и расширение

### Добавление нового источника

1. В админ-панели backend (или напрямую в БД) добавьте запись в `data_sources`:
   - Тип: telegram/vk/website/rss
   - ID канала/группы или URL
   - Регион и категория
   - Интервал обновления (минуты)

2. Источник автоматически включается в ротацию парсинга

### Добавление правила алерта

Создайте запись в `parsing_rules`:
- Тип: alert
- Условия: ключевые слова, тональность, минимальный engagement
- Действия: severity, уведомление

### Интеграция с внешними API

Для VK API (улучшенный парсинг):
1. Получите access token: https://vk.com/dev
2. Добавьте в `.env`: `VK_ACCESS_TOKEN=ваш_токен`
